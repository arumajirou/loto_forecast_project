# 22. NeuralForecast リソース解析 運用Runbook

## 1. 使い方（標準フロー）
1. ①概要で悪化指標（時間/失敗/GPU/DB）を特定する。
2. ②ランキングで悪化runを選ぶ。
3. ③ボトルネックで支配stageとDB比率を確定する。
4. ④エラーで例外型と直前イベントを確認する。
5. ⑤比較・検定で改善前後または群差を数値確認する。
6. ⑥提案に従って対策を実施する。

## 2. 代表アクション
- DB比率高: SQL見直し、Index最適化、バルク書込、TX見直し。
- GPU util低: 前処理/I/O/データ供給を最適化。
- 失敗率高: error_type別に再現条件を固定し、ガードを追加。
- stage偏在: 支配stageの分割、並列化、キャッシュ戦略を適用。

## 3. 定着ルール
- 週次で KPI と異常run 件数を確認。
- 変更後は必ず比較・検定で差を記録。
- 対策と効果を run_id 単位で運用ノートに残す。
